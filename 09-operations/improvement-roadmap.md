# Дорожная карта улучшения Brand OS (30 задач)

Этот документ содержит план трансформации текущего прототипа Brand OS в полноценную рабочую систему управления личным брендом.

## Фаза 1: Логика и Входящие (The "Chat")

1. **Задача 1: Поле быстрого ввода на дашборде.** Добавить текстовое поле в Sidebar для ввода идей, которые будут сохраняться в `08-inbox/ideas.md`.
2. **Задача 2: CLI-инструмент `add-task`.** Создать скрипт для добавления задач через терминал с автоматической простановкой даты.
3. **Задача 3: Просмотр Inbox в UI.** Создать отдельную вкладку на дашборде для чтения и разбора входящих идей.
4. **Задача 4: Система тегирования задач.** Добавить возможность помечать входящие тегами (#youtube, #philosophy) прямо при вводе.
5. **Задача 5: Динамическая статистика.** Переписать `app.js`, чтобы цифры "Выпущено" и "Запланировано" считались автоматически из JSON.

## Фаза 2: Разделение контента (Released vs Planned)

6. **Задача 6: Вкладки в YouTube-календаре.** Разделить таблицу на "Архив" (уже вышло) и "Пайплайн" (в работе).
7. **Задача 7: Визуализация прогресса.** Добавить индикатор выполнения серии (например, "4/50 выпусков готово") на главную.
8. **Задача 8: Статусы жизненного цикла.** Внедрить четкую цепочку статусов: `backlog` -> `scripting` -> `filmed` -> `editing` -> `released`.
9. **Задача 9: Виджет "Ближайший выпуск".** Вывести на Overview карточку с информацией только по самому ближайшему выпуску.
10. **Задача 10: Фильтр по рубрикам.** Добавить возможность быстро отфильтровать только философские или только бизнесовые темы.

## Фаза 3: Применение Брендбука (Enforcement)

11. **Задача 11: CSS-переменные бренда.** Перенести все Hex-коды из брендбука в CSS-переменные `:root`.
12. **Задача 12: Типографика по канону.** Подключить и применить шрифты IBM Plex Sans и JetBrains Mono.
13. **Задача 13: Тёмная тема "Deep Ink".** Сделать тёмный режим основным согласно цветовой схеме бренда.
14. **Задача 14: Авто-проверка текстов (Linter).** Скрипт, который подсвечивает использование запрещенных слов из брендбука (инфобизнес-лексика).
15. **Задача 15: Галерея референсов.** Интегрировать визуальные мудборды в раздел "Ассеты".

## Фаза 4: Операционное управление (EMS)

16. **Задача 16: Контроль WIP (Work In Progress).** Предупреждение на дашборде, если в работе одновременно более 3 выпусков.
17. **Задача 17: Расчет Takt Time.** Автоматический расчет среднего времени между выпусками для соблюдения графика.
18. **Задача 18: Журнал изменений в UI.** Отображать последние записи из `change-log.md` прямо на пульте управления.
19. **Задача 19: Чек-лист Definition of Done.** Всплывающий список обязательных проверок перед переводом выпуска в `released`.
20. **Задача 20: Автоматизация коммитов.** Чтобы каждое действие через дашборд создавало чистый Git-коммит.

## Фаза 5: Инструменты продакшена

21. **Задача 21: Режим телесуфлера.** Кнопка, разворачивающая текст сценария на весь экран крупным шрифтом.
22. **Задача 22: Генератор постов для Instagram.** Авто-создание черновика поста на основе тезисов YouTube-выпуска.
23. **Задача 23: Каталог фото с фильтрами.** Возможность фильтровать фото по тегам `approved`, `needs_validation`.
24. **Задача 24: Карта философов.** Визуальный граф связей между философскими концептами и бизнес-задачами.
25. **Задача 25: Интеграция с YouTube API.** Подгрузка реальных просмотров и лайков для вышедших видео.

## Фаза 6: Полировка и UX

26. **Задача 26: Мобильная версия.** Адаптация дашборда для удобного использования со смартфона на съемках.
27. **Задача 27: Глобальный поиск 2.0.** Поиск по всем файлам системы (философы, задачи, правила).
28. **Задача 28: Экспорт "Пакета выпуска".** Одна кнопка для скачивания всех материалов к выпуску (сценарий, обложка, теги).
29. **Задача 29: Мониторинг здоровья системы.** Проверка наличия всех обязательных файлов и структур по `schema.md`.
30. **Задача 30: Интерактивный онбординг.** Мини-инструкция при первом запуске дашборда.
