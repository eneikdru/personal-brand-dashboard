# 09. DAU, WAU, MAU: активная аудитория

## Цель выпуска
Зритель научится выбирать active event и период активности под модель продукта.

## Хук
MAU вырос на 30%, но денег и retention нет. Возможно, вы считаете активностью просто открытие приложения.

## Структура выпуска на 10 минут
| Время | Блок | Что дать зрителю |
|---|---|---|
| 0:00-0:40 | Ситуация с собеседования | MAU вырос на 30%, но денег и retention нет. Возможно, вы считаете активностью просто открытие приложения. |
| 0:40-1:30 | Что отвечает метрика | Как измерять размер живой аудитории продукта без превращения метрики в vanity? |
| 1:30-2:40 | Простое определение | Объяснить метрику через поведение пользователя и бизнес-решение. |
| 2:40-4:00 | Формула | Показать формулу из `formula.md`, явно назвать числитель, знаменатель и период. |
| 4:00-5:30 | MVP-кейс | Для MVP редактора презентаций active user - тот, кто создал или экспортировал документ, а не просто открыл главную страницу. |
| 5:30-7:00 | Как измерять | Разобрать события, сегменты, окно наблюдения и качество данных. |
| 7:00-8:30 | Нюансы и ловушки | Показать 3-4 типичные ошибки, из-за которых метрика обманывает. |
| 8:30-9:30 | Интервью-блок | Дать короткий шаблон ответа: business model -> event -> formula -> segment -> decision. |
| 9:30-10:00 | Практика | Домашнее задание и один вопрос зрителю в комментарии. |

## Мини-сценарий
1. Начать с конкретной бизнес-ситуации, а не с определения.
2. Объяснить, какое решение принимает команда по этой метрике.
3. Разобрать формулу на числитель, знаменатель, период и сегмент.
4. Показать, как метрика ломается при плохой инструментализации.
5. Закончить интервью-ответом на 45-60 секунд.

## Кейс для доски или экрана
Для MVP редактора презентаций active user - тот, кто создал или экспортировал документ, а не просто открыл главную страницу.

## Домашнее задание
Для 5 продуктов написать, что считать active user и почему login не подходит.

## Вопросы для тренировки интервью
- Какой active event выбрать для fintech, marketplace и SaaS?
- Почему рост MAU может быть плохим сигналом?
- Как посчитать resurrected users?
